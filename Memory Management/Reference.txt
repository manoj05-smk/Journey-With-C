Def:
   Memory management is the process of handling how much memory a program uses through allocation, reallocation and deallocation.
   When you create a basic variable, C will automatically reserve space for that variable.


Memory Layout of a C Program:
   ->When you run a C program, the OS gives it memory divided into 4 sections:
  
  HIGH ADDRESS  ┌─────────────────────┐
                │       STACK         │  ← local variables, function calls
                │   (grows downward)  │
                ├─────────────────────┤
                │         ↓           │
                │      (gap)          │
                │         ↑           │
                ├─────────────────────┤
                │       HEAP          │  ← malloc/calloc/realloc
                │   (grows upward)    │
                ├─────────────────────┤
                │       BSS           │  ← uninitialized global/static vars
                ├─────────────────────┤
                │       DATA          │  ← initialized global/static vars
                ├─────────────────────┤
 LOW ADDRESS    │       TEXT          │  ← your compiled code (read-only)
                └─────────────────────┘

Allocate Memory:
->The process of reserving memory is called allocation.
->C has two types of memory: Static memory & dynamic memory.

Static Memory:
->Static memory is memory that is reserved for variables before the program runs.
->Allocation of static memory is also known as compile time memory allocation

Dynamic Memory:
->Dynamic memory is memory that is allocated after the program starts running. 
->Allocation of dynamic memory can also be referred to as runtime memory allocation.
->Dynamic memory does not belong to a variable, it can only be accessed with pointers.

use the malloc() or calloc() functions For Allocation. 
Note: include the <stdlib.h> header to use those functions.

Allocation of Dynamic Memory:
1. using malloc(): => Initialize Garbage Values  // allocates 'size' bytes, returns void*
     ptr = (type*) malloc(size_in_bytes); ||  Eg: int *p = (int*) malloc(sizeof(int));
Note:
---------------------------------------------------------------------------
    // Imagine you change int → double later
    // Style A — WITH cast (you must update in 2 places!)
    double *arr = (double*)malloc(n * sizeof(double));    <==
    //            ^^^^^^^^ forgot to change this cast? silent bug!
    // Style B — WITHOUT cast (only 1 place to change)
    double *arr = malloc(n * sizeof(double));             <==
    // Clean, no redundancy 
---------------------------------------------------------------------------
   // The most robust pattern used by professionals:
   int    *arr = malloc(n * sizeof(*arr));
   double *arr = malloc(n * sizeof(*arr));
   char   *arr = malloc(n * sizeof(*arr));
   //                        ^^^^^^^^^
   //              sizeof automatically follows the pointer type!
   //              Change the type on the left → right side auto-adapts!
----------------------------------------------------------------------------

2. using calloc(): => Initialize Zeros  // allocates count*size bytes, ALL set to ZERO
     ptr = (type*) calloc(n, size);       ||  Eg: int *p = (int*) calloc(5, sizeof(int));

Reallocate Memory:
->If the amount of memory you reserved is not enough, you can reallocate it to make it larger.
->Reallocating reserves a different (usually larger) amount of memory while keeping the data that was stored in it.

Usage: realloc() function takes two parameters:
int *ptr2 = realloc(ptr1, size);
           * The first parameter is a pointer to the memory that is being resized.
           * The second parameter specifies the new size of allocated memory, measured in bytes.

Deallocate (free) Memory:
->Deallocation is also referred to as "freeing" the memory.
->Dynamic memory stays reserved until it is deallocated or until the program ends.

Usage:
* To deallocate memory, use the free() function:
        free(pointer);

Memory Leaks:
 A memory leak happens when dynamic memory is allocated but never freed.

Note: It is considered a good practice to set a pointer to NULL after freeing memory so that you cannot accidentally continue using it.
Eg:
   free(ptr);
   ptr = NULL;
